# ConvertFiles Distribution Playbook

## Principle
Every landing page should become a distribution asset, not just a URL that sits in the sitemap.

## Batch posting rhythm
- Publish the pages.
- Submit the sitemap if the batch is large.
- post 1 short LinkedIn update
- post 1 X thread
- post 1 relevant community post when there is a real fit

## Example post angles
- Image workflow pages:
  - "Made a privacy-first guide for reducing image size for websites without uploading files."
  - "If WebP keeps breaking your design workflow, this WebP-to-PNG guide solves the exact handoff problem."
- Bangla workflow pages:
  - "Built a browser-based Bijoy to Unicode workflow for old Bangla text that needs to work on modern devices."
  - "If your Illustrator or Photoshop workflow still depends on Bijoy, these new Bangla guides may help."
- Developer pages:
  - "Quick explanation of JSON formatting vs validation plus a browser-based formatter for fast payload cleanup."

## Outreach target list
- browser-tool roundup sites
- privacy-first tool lists
- Bangla publishing communities
- Bangladesh design and DTP groups
- developer utility roundups
- Shopify and WordPress image-optimization communities

## Outreach rule
Pitch the solved problem, not the homepage.

## SaaSHub submission notes
- Use the privacy-first angle.
- Mention browser-based processing.
- Link both the homepage and one strong workflow page.

## Review metric after each batch
- impressions
- CTR
- tool starts
- tool completions
- next-tool clicks
